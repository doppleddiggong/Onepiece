다음 Git diff를 기반으로 커밋 메시지를 생성하세요.

출력 형식은 반드시 아래 구조를 따릅니다.

1) 제목  
   - Conventional Commit 형식으로 1줄 작성  
   - 예시: feat: 새로운 로딩 처리 로직 추가

2) 문제·의도 설명 (1단락)  
   - 이번 변경을 하게 된 배경과 목적을 자연스럽게 설명  
   - 과도하게 딱딱한 문장 금지  
   - 문장 길이는 너무 짧게 끊지 말고 자연스러운 흐름 유지  

3) 변경 사항 요약 (bullet 목록)  
   - 기능 관점에서 핵심 변경을 정리  
   - 클래스명·함수명은 필요한 경우에만 언급  
   - 파일 경로, 라인 수, 통계(+/-)는 포함하지 않음  
   - 각 bullet은 간결하게 작성

톤 가이드:
- 토스 기술블로그처럼 단정하고 읽기 좋은 기술 서술로 작성할 것  
- 과한 격식·문서체 금지  
- 불필요한 강조나 감탄 표현 금지  
- "무엇을·왜·어떻게" 흐름이 부드럽게 이어지도록 작성

출력 구조 예시:

fix: 문제 요약

문제의 배경과 수정 이유를 2~3문장 정도로 자연스럽게 설명합니다.

- 변경 1  
- 변경 2  
- 변경 3
