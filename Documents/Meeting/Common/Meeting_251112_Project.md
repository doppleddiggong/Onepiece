# AI 파트 회의록 요약 (2025-11-12)

## 🧭 회의 주제
AI 파트의 개발 방향을 “기능 구현 중심”으로 할 것인지, 
“교육 커리큘럼 중심”으로 할 것인지 결정하기 위한 논의0

---

## 1️⃣ 주요 논점 정리

| 구분 | 내용 |
|------|------|
| **논의 배경** | 프로젝트 초기 단계에서 AI 파트의 목표를 정해야 함. “교육형 콘텐츠”로 갈지, “AI 기능 프로토타입”을 우선 개발할지 방향 모색. |
| **의문 제기** | “실제 교육 서비스를 구현할 것인가?” vs “우선 기능 단위로 완성할 것인가?” |
| **예시 제시** | ① 사용자가 칠판에 ‘ㄱ’을 그리면 AI가 인식 후 ‘맞음/틀림’을 판단해 피드백 제공.<br>② 음성으로 발음을 입력해 평가하는 기능 등. |
| **의견 ① (기능 중심)** | 글자 인식/판별 등 구체적이고 측정 가능한 기능 단위로 먼저 개발.<br>→ 커리큘럼은 기능 안정화 이후 설계. |
| **의견 ② (교육 중심)** | “글자 쓰기 → 발음하기 → 문장 구성하기” 등 학습 흐름을 먼저 정의하고 이에 맞춰 AI 기능을 구성. |
| **현 단계 결론** | 오전 논의에서 제시된 ‘자음·모음 조합으로 글자 만들기’ 기능을 우선 구현 대상으로 검토 중.<br>즉, **‘가상 글자 생성 및 판별 기능’**이 1차 개발 후보로 확정되는 분위기. |

---

## 2️⃣ 결정 및 후속 과제

| 항목 | 내용 |
|------|------|
| **결정 사항** | 기능 중심 접근으로 시작 — “글자 인식/판별” AI 기능을 프로토타입으로 구현. |
| **후속 논의 필요** | - 교육 커리큘럼 설계 시점 결정 (기능 안정화 이후)<br>- 판별 정확도, 입력 방식(마우스, 터치, 펜), 데이터셋 확보 등 세부 설계 필요 |
| **참고 방향** | - 이미지 기반 입력 처리 (필기 인식)<br>- 결과 반환 및 피드백 루프 설계<br>- 추후 발음 평가, 문장 학습 등 단계적 확장 고려 |

---

## ✅ 요약 결론
팀은 “글자 인식 및 판별 기능”을 중심으로 **AI 프로토타입을 우선 구현**하고, 기능이 안정화된 이후 **교육 커리큘럼(학습 플로우)** 확장으로 방향을 잡음.
