# Silver Care App — 프로젝트 회의록 (2025-11-10)

## 1. 개요
- **일자:** 2025-11-10
- **프로젝트:** Silver Care App
- **회의 목적:** 노인 복지와 AI 기술을 결합한 서비스 방향 및 멀티플레이 구조 검토
- **참석:** 기재 없음
- **키워드:** 노인 복지, AI, 멀티플레이, MediaPipe

---

## 2. 프로젝트 진행 현황 (상위 레벨)
### 2.1. 진행 상황 요약
- 팀은 노인 복지, 교육, 게임 요소를 하나로 묶은 AI 기반 실버 케어 앱을 1차 콘셉트로 확정하고, 차기 회의까지 기능 세분화와 기술 검증을 준비한다.

### 2.2. 주요 변경 사항(Architecture / Scenario / AI / DevOps / 클라이언트 등)
- 접근성과 지속성을 우선하는 방향으로 결과물을 설계하며, MediaPipe + STT/TTS + RPC 분리 구조를 중심으로 기술 아키텍처를 조정한다.

### 2.3. 차주 또는 마일스톤 대비 위치
- 컨셉 확정 단계에서 기능 정의와 RPC/AI 성능 검증 계획 수립 단계로 이동함.

---

## 3. 논의 내용 (세부)
### 3.1. 기획 / 시나리오
- 치매 예방 게임, 가정용 재활·운동 보조, 손주 AI, 동화책 자동 생성, 보이스피싱 교육 등 다양한 웰니스 콘텐츠를 연계한 시나리오 가능성을 타진.

### 3.2. 시스템 / 아키텍처 (UE5 / 서버 / AI 구조 포함)
- RPC 기반 멀티플레이를 일반 통신과 분리하고 MediaPipe 기반 모션 인식, STT/TTS 및 AI 생성 이미지(Lora·SD) 등을 포함한 시스템 구조로 설계.

### 3.3. 기술 구현 논의 (UE5·AI·DevOps 통합)
- STT→TTS 피드백 루프, RPC 네트워크 분리, 실버 세대 행동 데이터 구조화 및 재학습 모델 등 기술 검증 항목을 정의.

### 3.4. 디자인 / UX / 리소스
- 접근성 높은 UI/UX와 실버 세대에 맞춘 대화형 튜터, 사용성 중심의 콘텐츠 흐름을 디자인 지향.

### 3.5. 일정 / 리스크 / 의사결정
- 다음 회의까지 기능 세부화와 기술 실험을 완료하고, 접근성·성능·데이터 확보 리스크를 계속 모니터링.
