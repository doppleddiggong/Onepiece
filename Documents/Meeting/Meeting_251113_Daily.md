## 2. 오늘 논의 Summary

### 2.1 팀 온보딩(OT) 운영 방식

* 신규 멤버는 **무한도전·황금어장식 캐릭터 구축** 방식처럼,
  일정 기간 실제 작업을 굴리며 **강점·역할·성향 파악 과정**이 필요.
* 이번 프로젝트도 마찬가지로 **OT 없이 바로 투입하면 사고 발생 → 반드시 OT 진행**.

### 2.2 작업 문화 및 일정 습관

* 대부분 팀원은 **밤 11시까지 작업**하는 문화가 형성되어 있음.
* 정규 시간 이후에도 작업이 이어지므로, **오해 방지를 위해 재택 작업 시 작업 화면을 켜두는 등 투명성 유지**.

### 2.3 주말 일정

* 금~일 일정:

  * 금요일 약속 이후 → 토요일 오후 작업 시작
  * 토·일 풀타임으로 **기획서/시나리오/클라이언트 구성 정리 진행 예정**

## 3. 시나리오/콘텐츠 방향성

### 3.1 최종 플레이 시나리오

* 외국인이 한국에 입국하여 다음 절차를 수행하는 구조:

  1. 비행기 → 입국장 도착
  2. 줄 서기 → 입국 심사
  3. 택시 이동
  4. 호텔 체크인
  5. 식사 주문
  6. 관광

### 3.2 러닝 목표(읽기·쓰기·말하기·듣기)

* **읽기(Read):** 표지판·안내문 읽기
* **쓰기(Write):** 글씨 쓰기 / 입력 기반 (치훈)
* **듣기·말하기(Speak):** 입국 심사, 주문 상황 등 (주백)

### 3.3 캐릭터 / 아트 방향

* **플레이어 캐릭터:** 아스트로봇 계열(작고 귀여운)
* **NPC:** 주토피아 경찰서장의 큰 체구 느낌
* 외형/품질 기준(캐릭터 품, 디자인 품) 정리 예정

## 4. 기술·구현 논의

### 4.1 토이 프로젝트 Flow(시작~끝 턴)

* 현재 팀원이 **토이 프로젝트 Flow**를 만들고 있는데,
  이것은 **우리 프로젝트에 직접 반영된 내용은 아님**.
* 목적:

  * 전체 게임의 한 턴(시작~종료)을 설계하는 경험
  * 이 경험을 바탕으로 **본 프로젝트의 구조 샘플**을 도출
* 오늘 오후부터는 **본 프로젝트 Flow로 전환** 예정.

### 4.2 클라이언트 개발 방향

* **오늘 오전:**

  * 어제 회의된 요구사항을 바탕으로 **상황 시퀀스/씬 플로우 작성**
  * 캐릭터 외형/역할 정리
* **오늘 오후:**

  * 커밋 메시지 자동화(주백) 시연 및 전체 팀 세팅
  * 각자 로컬 PC 세팅 후 사용법 공유
  * 클라이언트/서버 간 API 스펙 정리 필요

### 4.3 글쓰기 기능(그림판 스타일)

* 마우스로 글씨 쓰는 기능 조사
* 캐릭터 색상 선택/변경
* 간단한 UI까지 프로토타입 구축 예정

### 4.4 이미지/음성 전송 구조

* **Base64 통일**

  * JPG 등 이미지 → Base64
  * 음성(STT/TTS) → Base64
* 서버 API는 Base64 기반으로 설계
* 웹기반 문서/프로토타입 제작 후 전달 예정

## 5. AI 팀 협업 / 기술 설계 방향

### 5.1 AI 팀 기술 조사 범위

* GPT 기반뿐 아니라 **한국어 특화 로컬 모델** 탐색
* RAG(VectorDB) 구조 설계 병행
* 월요일 전까지 **기술 검토 + 대략적 설계안** 준비

### 5.2 시스템 설계가 포함해야 하는 것

* **데이터 흐름 및 통신 구조**

  * Base64 전송
  * Whisper STT / Coqui TTS 처리
  * LLM 프롬프트 구조
* **API 프로토콜 문서**

  * 텍스트 입력 → 분석
  * 음성 입력 → 텍스트 변환
  * 클라이언트 행동 피드백

### 5.3 RAG의 적용 범위

* RAG는 **법률·규정·입국 심사 조건** 등 변동성이 큰 데이터 관리에 적합
* 가게 정보 등은 **RAG보다 웹서치가 적합**
* RAG 데이터는 **VectorDB(embedding 기반)**

  * 텍스트 → embedding
  * 쿼리 → embedding
  * 유사도 기반 검색
* 기업/기관 데이터의 경우 보안상 로컬 RAG 필요성이 큼
